How do I call my route?

The Subroute.io execution engine uses convention to determine which method of your class to call. When making a RESTful request to the following endpoint.

The Subroute.io execution engine will use the HTTP verb (e.g. GET, POST, DELETE, etc...) to determine which method to call. Subroute.io will grab the first method whose name begins with the HTTP verb and execute it by passing in the current request's headers, query string, and body.

If your code intends to return a result, it should instantiate an ExecutionResponse and set its status code, headers, and payload to return to the caller.
