How do I call my route?

The Subroute.io execution engine uses convention to determine which method of your class to call. When making a RESTful request to the following endpoint.

The Subroute.io execution engine will use the HTTP verb (e.g. GET, POST, DELETE, etc...) to determine which method to call. Subroute.io will grab the first method whose name begins with the HTTP verb and execute it by passing in the current request's headers, query string, and body.

If your code intends to return a result, it should instantiate an RouteResponse and set its status code, headers, and payload to return to the caller.

You can also use the base methods to make returning a result easier. The following methods are available:

Serialize a POCO object as JSON and return 200 OK.
Json(HttpStatusCode, object)

Serialize a POCO object as XML and return 200 OK.
Xml(HttpStatusCode, object)

Return provided string and return 200 OK.
String(HttpStatusCode, string)

Use the provided stream as the response payload and return 200 OK.
Stream(HttpStatusCode, Stream, string)

Return an empty result with 204 No Content.
NoContent()

Return an empty result with 400 Bad Request.
BadRequest()

Return an empty result with the specified status code.
StatusCode(HttpStatusCode)

Use the specified response formatter to serialize the provied payload and return the specified status code.
Formatted(HttpStatusCode, string, object)
